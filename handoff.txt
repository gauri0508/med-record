⬜ Step 6: env/environment.py — CORE ENVIRONMENT CLASS
   ---------------------------------------------------------
   This is the most important file. It's the "game engine."

   Class: MedRecordAuditEnv

   Methods needed:

   reset(difficulty, case_id=None):
     - Load a random case from data/cases/{difficulty}/
     - Hide ground_truth_issues from the agent
     - Return initial state: patient info + record_index (just id, date, type, summary — NOT full content)
     - Set budget based on difficulty (easy=15, medium=25, hard=30)
     - Reset step counter, findings list, reviewed records

   step(action):
     - Parse action JSON
     - Handle 4 action types:
       a) read_record(record_id) → return full record content, decrement budget
       b) cross_reference(query) → search drugs.json/diseases.json/lab_ranges.json, return relevant info
       c) flag_issue(type, description, evidence) → store finding, decrement budget
       d) submit_report() → end episode, compute reward, return final score
     - Return: {state, reward, done, info}

   state():
     - Return current state (what agent has seen so far)

   Key data structures:
   - self.case: loaded case JSON
   - self.patient: patient info (visible to agent)
   - self.records: all records (hidden, revealed on read_record)
   - self.record_index: [{id, date, type, summary}] (visible to agent)
   - self.reviewed: list of record IDs agent has read
   - self.findings: list of issues agent has flagged
   - self.ground_truth: hidden answer key
   - self.budget: remaining steps
   - self.done: episode over?


⬜ Step 7: env/reward.py — REWARD FUNCTION
   ---------------------------------------------------------
   Computes score (0.0 - 1.0) for agent's findings.

   Two scoring methods:

   1. Programmatic scoring:
      - Compare agent's flag_issue type with ground_truth type → type match?
      - Compare agent's evidence records with ground_truth evidence → overlap?
      - If type matches AND evidence overlaps → correct finding

   2. LLM scoring (for description similarity):
      - Compare agent's description with ground_truth description
      - Use semantic similarity (or simple keyword overlap for now)

   Formula:
     findings_score = sum of per-finding rewards (capped at 0.7)
       - Correct critical finding: +0.25
       - Correct moderate finding: +0.15
       - Correct minor finding: +0.10
       - False positive: -0.10
     efficiency_bonus = (budget_remaining / total_budget) * 0.15
     completeness_bonus = (correct_findings / total_findings) * 0.15
     TOTAL = clamp(findings_score + efficiency_bonus + completeness_bonus, 0.0, 1.0)


⬜ Step 8: env/graders.py — TASK GRADERS
   ---------------------------------------------------------
   3 grader functions that run a case and validate the score.

   easy_grader(env):
     - Run one easy case
     - Verify score is 0.0-1.0
     - Check that env properly handles all actions

   medium_grader(env):
     - Same for medium cases

   hard_grader(env):
     - Same for hard cases

   Each grader:
   1. Calls env.reset(difficulty)
   2. Simulates a simple agent (read a few records, flag obvious issues)
   3. Calls env.step(submit_report)
   4. Verifies the returned score is 0.0-1.0


⬜ Step 9: env/server.py — FASTAPI SERVER
   ---------------------------------------------------------
   Wrap the environment in HTTP endpoints.

   Endpoints:
     POST /reset  → body: {difficulty, case_id?} → returns initial state
     POST /step   → body: {action, ...params} → returns {state, reward, done, info}
     GET  /state  → returns current state
     GET  /health → returns {"status": "ok"} (HTTP 200)

   Uses Pydantic models for request/response validation.
   Runs on port 7860 (required for HF Spaces).


⬜ Step 10: inference.py — BASELINE AGENT
   ---------------------------------------------------------
   Root-level file. Uses OpenAI Client for LLM calls.

   Environment variables needed:
     API_BASE_URL — LLM endpoint
     MODEL_NAME — model identifier
     HF_TOKEN — Hugging Face authentication

   Simple strategy:
   1. Call /reset
   2. Read the record_index
   3. Ask LLM: "which records look most important to read?"
   4. Read those records via /step(read_record)
   5. Ask LLM: "any issues in what you've seen?"
   6. Flag issues via /step(flag_issue)
   7. Submit report via /step(submit_report)

   CRITICAL: Must print structured stdout logs:
     [START] {"task": "easy_001", "timestamp": "..."}
     [STEP]  {"action": "read_record", "record_id": 3, "step": 1, "timestamp": "..."}
     [STEP]  {"action": "flag_issue", "type": "drug_interaction", "step": 2, "timestamp": "..."}
     [END]   {"score": 0.85, "steps": 5, "timestamp": "..."}

   Format must be EXACT — any deviation = wrong evaluation score.
