Files created:

1. data/lab_ranges.json
   ---------------------------------------------------------
   WHAT IT IS:
   Reference database of normal/abnormal ranges for 50+ lab tests.

   WHY IT'S NEEDED:
   - When the environment generates lab results for patient records,
     it uses these ranges to mark values as HIGH/LOW/NORMAL
   - The "cross_reference" action uses this to tell the agent
     what's normal vs abnormal
   - Graders use it to verify if the agent correctly identified
     abnormal lab trends

   WHAT'S INSIDE:
   13 categories of lab tests:

   a) Hematology (18 tests)
      - CBC: WBC, RBC, hemoglobin, hematocrit, platelets
      - Differentials: neutrophils, lymphocytes, monocytes, eosinophils, basophils
      - Other: MCV, MCH, MCHC, RDW, ESR
      - Coagulation: INR, PT, aPTT

   b) Basic Metabolic Panel (10 tests)
      - Electrolytes: sodium, potassium, chloride, bicarbonate
      - Kidney: BUN, creatinine, eGFR
      - Other: glucose (fasting & random), calcium
      - eGFR has staging thresholds (normal > 90, mild 60-89, moderate 30-59, severe 15-29, failure < 15)

   c) Comprehensive Metabolic Panel (7 tests)
      - Liver: ALT, AST, ALP, bilirubin (total & direct)
      - Protein: total protein, albumin

   d) Endocrine (7 tests)
      - Diabetes: HbA1c (with prediabetic/diabetic thresholds)
      - Thyroid: TSH, free T4, free T3
      - Other: cortisol, fasting insulin, vitamin D

   e) Lipid Panel (4 tests)
      - Total cholesterol, LDL, HDL, triglycerides
      - Includes risk thresholds (optimal, borderline, high)

   f) Cardiac Markers (7 tests)
      - Troponin I, BNP, CK, CK-MB, LDH, CRP, homocysteine
      - Includes critical thresholds for acute events

   g) Kidney-specific (3 tests)
      - Uric acid, urine microalbumin, albumin-creatinine ratio

   h) Liver-specific (2 tests)
      - GGT, ammonia

   i) Iron Studies (4 tests)
      - Serum iron, TIBC, ferritin, transferrin saturation

   j) Autoimmune (4 tests)
      - ANA, RF, anti-CCP, anti-dsDNA

   k) Vitamins (2 tests)
      - B12, folate

   l) Urinalysis (5 tests)
      - pH, specific gravity, protein, glucose, blood

   KEY FEATURE: Gender-specific ranges where applicable (e.g., hemoglobin,
   creatinine, ferritin) and critical values that indicate life-threatening
   levels (e.g., potassium < 2.5 or > 6.5)


2. data/drugs.json
   ---------------------------------------------------------
   WHAT IT IS:
   Database of 26 commonly prescribed drugs with their interactions,
   contraindications, and monitoring requirements. Plus allergy
   cross-reactivity data.

   WHY IT'S NEEDED:
   - Patient cases reference these real drugs with real dosages
   - The "cross_reference" action uses this to return drug interaction
     info when the agent queries it
   - Ground truth issues are based on real drug interactions from
     this database (e.g., "warfarin + aspirin = major bleeding risk")
   - Graders validate that flagged drug interactions actually exist

   WHAT'S INSIDE:

   26 drugs organized by:
   - name: Real drug name
   - class: Drug class (e.g., "ACE Inhibitor", "Statin")
   - indication: What it treats
   - common_doses: Real-world dosing
   - contraindications: When NOT to prescribe (e.g., "metformin if eGFR < 30")
   - warnings: Side effects and risks
   - interactions: Drug-drug interactions with severity (major/moderate/contraindicated)
   - monitoring: What labs to check and how often
   - rxnorm: Real RxNorm code for each drug

   Drug categories covered:
   - Diabetes: metformin, insulin glargine
   - Cardiovascular: lisinopril, losartan, amlodipine, metoprolol,
     hydrochlorothiazide, furosemide, spironolactone, digoxin, amiodarone
   - Anticoagulation: warfarin, aspirin, clopidogrel
   - Cholesterol: atorvastatin
   - GI: omeprazole, pantoprazole
   - Pain: ibuprofen, tramadol, gabapentin
   - Antibiotics: amoxicillin, ciprofloxacin
   - Endocrine: levothyroxine, prednisone
   - Rheumatology: allopurinol
   - Psychiatry: sertraline

   Allergy cross-reactivity section:
   - Penicillin allergy → which drugs to avoid (amoxicillin, ampicillin)
   - Sulfonamide allergy → cross-reactive drugs
   - NSAID allergy → cross-reactive drugs and safe alternatives

   REAL-WORLD ACCURACY:
   All interactions sourced from FDA drug safety communications.
   Examples of real interactions included:
   - Warfarin + amiodarone: reduce warfarin dose by 30-50%
   - Omeprazole + clopidogrel: reduces antiplatelet effect (use pantoprazole)
   - Metformin + eGFR < 30: contraindicated (lactic acidosis risk)
   - Digoxin + amiodarone: increase digoxin levels by 70-100%


3. data/diseases.json
   ---------------------------------------------------------
   WHAT IT IS:
   Database of 15 diseases with real diagnostic criteria, symptoms,
   complications, and — most importantly — commonly missed issues.

   WHY IT'S NEEDED:
   - Patient cases are built around these diseases
   - The "common_missed_issues" field directly inspires the ground
     truth issues we hide in patient cases
   - Graders use diagnostic criteria to validate agent findings
   - The "cross_reference" action uses this for disease info

   WHAT'S INSIDE:

   15 diseases across 9 medical specialties:

   Endocrine:
   - Type 2 Diabetes Mellitus (E11)
   - Hypothyroidism (E03)

   Cardiovascular:
   - Essential Hypertension (I10)
   - Atrial Fibrillation (I48)
   - Heart Failure (I50)

   Renal:
   - Chronic Kidney Disease (N18)

   Hematology:
   - Iron Deficiency Anemia (D50)
   - Deep Vein Thrombosis / PE (I82/I26)

   Pulmonary:
   - COPD (J44)

   Rheumatology:
   - Gout (M10)
   - Rheumatoid Arthritis (M06)

   Musculoskeletal:
   - Osteoporosis (M81)

   Psychiatry:
   - Major Depressive Disorder (F33)

   Autoimmune:
   - Systemic Lupus Erythematosus (M32)

   Gastroenterology:
   - Celiac Disease (K90.0)

   Each disease includes:
   - ICD-10 code (real billing code)
   - Symptoms list
   - Risk factors
   - Diagnostic criteria (with real thresholds)
   - Key labs to order
   - Complications
   - Common medications
   - Monitoring schedule
   - COMMON MISSED ISSUES ← this is the most important field!
     These are real-world mistakes that doctors frequently make,
     sourced from medical literature. They become the hidden
     problems in our patient cases.

   Examples of common missed issues:
   - "declining eGFR not addressed" (CKD)
   - "metformin not adjusted for renal function" (Diabetes)
   - "iron deficiency anemia attributed to menstruation without GI workup" (Celiac)
   - "SSRI + tramadol serotonin syndrome risk" (Depression)
   - "anticoagulation not started despite high stroke risk" (AFib)

STEP 6-9: ENVIRONMENT CODE (COMPLETED)
========================================================

What was done:
- Built all 4 core Python files that make the environment work

Files created:

1. env/environment.py — CORE RL ENVIRONMENT
   - MedRecordAuditEnv class
   - reset(difficulty, case_id) → loads case + task.json, returns initial state
   - step(action) → handles 4 actions: read_record, cross_reference, flag_issue, submit_report
   - state() → returns current state (patient, records reviewed, findings, budget)
   - _compute_reward() → scores findings against ground truth
   - _match_finding() → matches agent's flags to ground truth (type + evidence + keywords)
   - Loads task instructions from tasks/{case_id}/task.json on reset

2. env/reward.py — REWARD FUNCTION
   - match_finding() → scores how well a finding matches ground truth (0.0-1.0)
     * Type match: 0.4 weight
     * Evidence overlap: 0.3 weight
     * Description keyword overlap: 0.3 weight
   - compute_reward() → final score calculation
     * findings_score (max 0.70): correct findings + false positive penalties
     * efficiency_bonus (max 0.15): budget remaining / total budget
     * completeness_bonus (max 0.15): correct findings / total issues
     * Returns 0.0 when no findings submitted

3. env/graders/ — INDIVIDUAL GRADERS (1 per case)
   - base.py: shared run_case() logic
   - 9 individual grader files: easy_001.py through hard_003.py
   - __init__.py: exports all graders + easy_grader/medium_grader/hard_grader
   - __main__.py: runs all 9 when you do: py -m env.graders
   - Each grader validates environment loads correctly for that case

4. env/server.py — FASTAPI SERVER
   - POST /reset → start episode (returns task + patient + record_index)
   - POST /step → take action (returns state + reward + done + info)
   - GET /state → current state
   - GET /health → returns 200 (required for HF Spaces)
   - Runs on port 7860


========================================================
STEP 10: INFERENCE AGENT (COMPLETED)
========================================================

File: inference.py (root level, required by hackathon)

What it does:
- Connects to the environment server via HTTP
- Uses OpenAI Client to call an LLM
- For each task:
  1. Calls /reset → gets task instruction + patient + records
  2. Sends task to LLM: "which records should I read?"
  3. Reads priority records via /step(read_record)
  4. Sends records to LLM: "what issues do you see?"
  5. Flags issues via /step(flag_issue)
  6. Submits report via /step(submit_report)
- Prints [START]/[STEP]/[END] structured logs (STRICT format)

Usage:
  py inference.py              → run ALL 9 tasks (what judges run)
  py inference.py easy_001     → run single task
  py inference.py easy         → run all easy tasks
  py inference.py hard         → run all hard tasks

Requires env vars: API_BASE_URL, MODEL_NAME, OPENAI_API_KEY (or use Ollama)

HOW THE FILES CONNECT
========================================================

1. tasks/{case_id}/task.json DEFINES:
   → What the AI agent should look for in each case

2. data/cases/{difficulty}/ CONTAINS:
   → Patient records + hidden ground truth issues (answer key)

3. env/environment.py LOADS:
   → Task instructions from tasks/
   → Patient cases from data/cases/
   → Reference databases from data/

4. env/reward.py SCORES:
   → Agent's findings against ground truth issues

5. env/graders/ VALIDATES:
   → Environment loads correctly, scores are 0.0-1.0

6. env/server.py WRAPS:
   → Environment in FastAPI endpoints (port 7860)

7. inference.py RUNS:
   → Connects to server, uses LLM to analyze records, flags issues
   → Prints [START]/[STEP]/[END] logs for judges
